# Tennoplan Design Context & System

## Design Context

### Users
**Endgame Warframe players** — grinding, theory-crafting, researching, building. They use Tennoplan as their **one-stop hub** for all tracking (relics, frames, weapons, content, progression). They're serious, focused, and expect professional information design.

**Usage Context**: 
- Quick glance (5 sec): Scan key information without focus
- Deep dive (2+ min): Explore detailed breakdowns, research, planning
- Works seamlessly across both contexts

### Brand Personality
**Polished. Informational. Clear. Consistent.**

Anti-references: card box garbage, generic monotone, neon-aggressive UIs (Alecaframe-style), cluttered dashboards.

The app should evoke: "This looks amazing. The text is easy to read. So presentable. Like those great infographics, but better. Even if I want detail, it's easy on the eyes. This is so comprehensive. The person who made this really understood what we need as players."

### Aesthetic Direction
**Professional information hub with Warframe authenticity.**

**Default Theme**: Dark teal/cyan base with gold accents
- Dark teal feels **tech-forward, Void-inspired, distinct**
- Gold accent supports **information hierarchy** (key details pop)
- Not generic. Memorable. Instantly recognizable as Tennoplan
- Like "Pixel Art" or "Warframe 1999"—a unique visual standard

**Future Themes**: Same system, different accent color. Gold → crimson steel path, cyan → ethereal void blues, etc. Same component library, themeable via CSS variables.

### Design Principles
1. **Information hierarchy is everything** — Key details jump out. Secondary info recedes. No visual noise.
2. **SaaS dashboard clarity** — Professional, focused, easy to scan. Not game-y.
3. **Consistent component system** — Reusable components maintain visual language across the app. Revisions kill momentum; design once, apply everywhere.
4. **Dark sophistication** — Dark background + gold + teal create visual depth and elegance without aggression.
5. **Soft blending where needed** — Colors complement each other through composition, not clash.

## Color System

### Base Palette (Dark Teal Theme)

**Primary Background**: `#0a1117` or similar (very dark, almost black, with cool undertone)
- Used for: Main background, surface-lowest
- Effect: Sophisticated, recedes, supports content

**Secondary Background**: `#161b22` (slightly lighter, dark)
- Used for: Cards, panels, content containers
- Effect: Visual separation without harsh contrast

**Accent - Gold/Tan**: `#d4af37` or `#e3c372` (existing)
- Used for: Borders, highlights, key information, emphasis
- Effect: Warmth, importance, guides attention
- Rule: Use sparingly (10-15% of visual weight)
- Key Detail: Key numbers, names, section titles get gold emphasis

**Accent - Teal/Cyan**: `#00d4ff` or muted equivalent (supporting accent)
- Used for: Secondary highlights, interactive states, progress
- Effect: Freshness, technical feeling
- Rule: Less frequent than gold; for specific interactions or secondary data

**Text - Primary**: `#e5e2e1` or lighter (light neutral)
- Used for: Body text, labels, interface text
- Good contrast on dark backgrounds

**Text - Muted**: `#a8a5a0` or similar (medium gray)
- Used for: Secondary information, labels, supporting text
- Still readable, clearly secondary

**Text - Highlight**: Gold (same as accent) or Teal (for interactive)
- Used for: Key stats, important names, emphasized information
- Creates hierarchy through color contrast

**Border/Dividers**: `#2d333b` or tinted to match theme
- Used for: Card borders, section separators, structural lines
- Effect: Organization without harshness

### Color Usage Rules
- **No pure black or pure white** — Always tinted toward the theme
- **Tint neutrals toward teal** — Even slight tint (0.005-0.01 chroma) creates cohesion
- **Gold is accent, not background** — Use it for emphasis, borders on cards, key numbers
- **Contrast supports hierarchy** — Key info pops (light on dark, gold on dark), secondary info is subtle
- **Functional color** — Red for alerts/warnings, green for success, these should be bright enough to read but not neon

## Typography System

### Font Families
- **Headlines**: Noto Serif (elegant, sophisticated, Orokin-inspired)
- **Body**: Inter or similar clean sans-serif (readable, professional)
- **Labels**: Same as body, but smaller, often muted color

### Type Scale (Fixed Sizes for Dashboard UI)
Use fixed `rem` scales for product UI, not fluid clamp sizing.

- **h1 (Display)**: 3.2rem / 51px — Page titles (e.g., "CELESTIAL PENDULUM MASTER HUB")
- **h2 (Large)**: 2.4rem / 38px — Section titles
- **h3 (Medium)**: 1.8rem / 29px — Card headers, subsection titles
- **h4 (Regular)**: 1.4rem / 22px — List headers, smaller cards
- **body**: 1rem / 16px — Main content, readable text
- **label**: 0.875rem / 14px — Field labels, secondary info
- **caption**: 0.75rem / 12px — Very small labels, metadata

**Line Height Inverse**: 
- Large text (h1, h2): 1.2 (tight)
- Medium text (h3, body): 1.5 (balanced)
- Small text (label, caption): 1.4 (readable despite size)

**Weight**:
- Bold (700): Headlines, key data, emphasis
- Regular (400): Body text
- Light (300): Secondary text, muted info

### Typography Rules
- **Hierarchy through size + weight** — Don't rely on color alone
- **Key numbers bold** — Stats, timers, important values
- **Labels muted** — Let the data shine
- **No all-caps body text** — Reserve for short labels/headers
- **~65-75ch max line length** — Prevents eye fatigue
- **Generous line height for light text** — Light text on dark needs breathing room

## Spacing System

### Modular Scale (4pt base)
```
4px   (--space-xs)
8px   (--space-sm)
12px  (--space-md)
16px  (--space-lg)
24px  (--space-xl)
32px  (--space-2xl)
48px  (--space-3xl)
64px  (--space-4xl)
```

### Component Spacing Rules

**Sections** (major content groupings):
- Space between sections: 48px or 64px (breathing room)
- Padding inside section: 32px or 48px
- Effect: Clear visual separation, hierarchy

**Cards/Panels** (information containers):
- Internal padding: 16px–24px (depends on content density)
- Space between cards in grid: 16px (tight, organized)
- Border: 1px, teal-tinted dark color or gold accent

**Within Cards** (tight organization):
- Icon + text spacing: 8px–12px
- Vertical spacing between items: 8px–12px
- Tight but organized; doesn't feel cramped

**Margins vs. Gaps**:
- Use `gap` for sibling spacing (eliminates margin collapse)
- Use margins for breathing room between sections
- Never use same padding everywhere

### Spacing Principle
**Varied spacing creates rhythm.** Generous space emphasizes sections, tight space organizes details. Don't be uniform.

## Component Patterns

### Sections (Major Content Groupings)
**Purpose**: Organize the page into distinct information domains

**Structure**:
```
Section Header (h2)
  └─ Description/subtitle (muted)
    └─ Content (cards, grids, lists)
```

**Styling**:
- Title: h2, bold, gold or text-highlight
- Space above: Extra margin (32px+) to emphasize importance
- Content padding: 24–32px
- Border/separator: Optional somatic line (gold gradient) above or below

**Example**: "CELESTIAL PENDULUM MASTER HUB" (title) → cycle sections below with generous space

### Cards (Information Containers)
**Purpose**: Group related information with clear boundaries

**Structure**:
```
Card
  ├─ Header (optional: icon + label)
  ├─ Primary content (large number, key info)
  ├─ Secondary content (details, list)
  └─ Footer (optional: CTA, metadata)
```

**Styling**:
- Background: Slightly lighter than page background (secondary bg color)
- Border: 1px teal-tinted or gold for emphasis
- Padding: 16–24px (depends on density)
- Radius: 6px (subtle, not rounded pill)
- Shadow: Minimal, just for depth

**Do**: Use cards for discrete information units (a timer, a mission, a stat)
**Don't**: Nest cards inside cards; flatten the hierarchy

### Grids (Card Layouts)
**Pattern**: `grid-template-columns: repeat(auto-fit, minmax(280px, 1fr))`
- Responsive, no breakpoints needed
- Each card ~280px minimum, expands to fill space
- Gap: 16px (consistent spacing)

**Usage**: Fissures, relics, missions, any repeating content

### Lists (Sequential Information)
**Structure**:
```
List Item
  ├─ Icon or number (leading visual)
  ├─ Title (bold, clear)
  └─ Meta/stat (muted, smaller)
```

**Styling**:
- No bullets; use whitespace
- Icon left-aligned or leading number
- Space between items: 8–12px (tight, organized)
- Hover state: Subtle background shift or border highlight

**Usage**: Challenges, stages, weapons, any sequential items

### Timers & Countdowns (Critical Information)
**Purpose**: Key information that demands attention

**Styling**:
- Large, bold typography (h2 or h3)
- Gold or bright accent color (stands out)
- Supporting label below (muted, smaller)
- Optional: Border or background tint to emphasize

**Example**:
```
1 DAY 23M
VOID FISSURE ACTIVE
```

### Hero Sections (Page Headers)
**Purpose**: Orient user, establish context

**Structure**:
```
Hero
  ├─ Title (h1, gold or bold)
  ├─ Subtitle (muted, context)
  └─ Optional: Hero image/background
```

**Styling**:
- Extra vertical padding (48–64px)
- Title color: Gold or text-highlight
- Subtitle: Muted
- Background: Optional cinematic image (subtle, not overwhelming)

**Do**: Use for major page sections (tabs)
**Don't**: Make it flashy or decorative; it should orient, not distract

## Information Hierarchy Rules

### The Priority System
1. **Glance (5 sec)**: Key metrics pop (large, bold, gold)
2. **Read (30 sec)**: Secondary info visible but quiet (smaller, muted)
3. **Deep (2+ min)**: Details accessible without clutter (nested, expandable if needed)

### Visual Hierarchy Techniques
- **Size**: Large = important (h1 > h3 > body)
- **Color**: Gold/highlight = important (muted = secondary)
- **Weight**: Bold = important (regular = body)
- **Position**: Top = important (bottom = less critical)
- **Whitespace**: More space = more important (tight = organized detail)
- **Icon/visual**: Leading visual guides attention (icon + text > text alone)

### Example: Challenge Card
```
[Icon]  NIGHTWAVE CHALLENGE         ← h3, bold, gold icon
        Complete 5 Void Fissures    ← body text, clear but not bold
        3/5 • 2 days remaining      ← muted, small, metadata
        [Progress Bar]              ← visual indicator
```

Key metric (3/5) isn't largest but is positioned and colored for scanning.

## Anti-Patterns (FORBIDDEN)

### ❌ DO NOT
1. **Side-stripe borders** on cards (`border-left: 3px+`) — Too generic, AI slop tell
2. **Gradient text** — Decorative noise, hard to read
3. **Glassmorphism everywhere** — Blur effects used carelessly distract
4. **Generic card grids** — Every card identical (icon + heading + text) — boring
5. **Nested cards** — Flattens visual hierarchy into noise
6. **Gray text on colored backgrounds** — Looks washed out; use shade of background
7. **Over-padding** — Same padding everywhere kills rhythm
8. **Bounce/elastic easing** — Feels dated; use ease-out (natural deceleration)
9. **Pure black or pure white** — Always tint toward the theme
10. **Neon accents on dark** — Cyber-aesthetic kill; use sophisticated colors

## Component Library (Reusable Elements)

### Foundation
- **Typography scales** (fixed sizes, line heights, weights)
- **Spacing scale** (4pt modular)
- **Color tokens** (background, text, accent)
- **Border radius** (subtle curves, no extreme pills)
- **Box shadows** (minimal, depth only)

### UI Components
- **Section Header** — h2 + optional subtitle + optional divider
- **Card** — Container + border + padding pattern
- **Button** (primary, secondary, ghost) — Consistent styling, hover states
- **Progress Bar** — Visual indicator for completion
- **Timer** — Large, bold, with supporting label
- **Icon + Text** — Consistent spacing pattern
- **List Item** — Icon + content + meta
- **Badge/Tag** — Small label with muted background
- **Divider** (somatic line) — Subtle gold gradient separator

### Interactive States
- **Hover**: Subtle border highlight or background shift
- **Focus**: Clear focus indicator (gold border or background)
- **Active**: Color change or weight shift
- **Disabled**: Reduced opacity, muted color

### Motion (Minimal, Purposeful)
- **Page transitions**: Fade in, 200–300ms
- **Card hover**: Subtle border color shift, 150ms ease-out
- **Progress bars**: Smooth fill animation, 500ms ease-out
- **Modals/overlays**: Fade + subtle scale, 200ms ease-out
- **Rule**: No bounce/elastic easing; use ease-out-quart or ease-out-quint for natural deceleration

## Theming Strategy (Future)

### How It Works
1. **CSS Variables** define the color system
2. **Same component library** works for all themes
3. **Theme = color variant**, not structural redesign
4. **Easy to add**: Just add new variable set, no code changes

### Theme Examples
- **Default (Dark Teal)**: Dark background, gold accent, teal secondary
- **Steel Path (Red)**: Dark background, deep red accent, black secondary
- **Ethereal (Void Cyan)**: Dark background, light cyan accent, muted purple secondary

**Implementation**: 
- CSS: `--color-primary: var(--theme-accent)`
- HTML: `<html data-theme="dark-teal">` or toggle via Zustand store
- Each theme has complete color set, no partial overrides

## Implementation Checklist

When building a new feature or tab:
- [ ] Follow spacing scale (no arbitrary padding)
- [ ] Use typography scale (no custom sizes)
- [ ] Apply color system (only use defined tokens)
- [ ] Information hierarchy: Does key info pop? Can secondary info recede?
- [ ] Responsive: Works on mobile/tablet/desktop
- [ ] Reuse components: Did you check the component library first?
- [ ] No anti-patterns: Review the forbidden list
- [ ] Accessibility: Contrast ratio, focus states, semantic HTML
- [ ] Motion: Purposeful, not decorative; ease-out easing

## Design Decisions Rationale

### Why Dark Teal/Cyan Base?
- **Distinctive**: Not generic gold (Orokin) — unique visual standard
- **Professional**: Tech-forward, SaaS-like, not "game UI"
- **Warframe-authentic**: Void/tech aesthetic, authentically Warframe
- **Hierarchical**: Dark recedes, gold/accent pops (info hierarchy)
- **Themeable**: Easy to swap colors later without structural changes
- **Memorable**: Like "Pixel Art" — instantly recognizable as Tennoplan

### Why Gold as Accent (Not Primary)?
- **Sparingly used** = high impact (accents work because they're rare)
- **Supports hierarchy** = key data jumps out
- **Not generic** = not just another gold-dark dashboard
- **Warm + Cool** = teal + gold creates visual depth and sophistication

### Why Component Library is Critical?
- **Consistency without revision** — Build once, use everywhere
- **Theming ease** — Swap colors via CSS variables
- **Faster shipping** — Copy-paste proven patterns
- **Scalability** — As features grow, components grow with them
- **Future-proof** — New tabs, new features follow the same system
